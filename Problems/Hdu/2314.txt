<h1 style="color:#1A5CC8">Dr. Podboq or: How We Became Asymmetric</h1>
<font><b><span style="font-family:Arial;font-size:12px;font-weight:bold;color:green">Time Limit: 6000/2000 MS (Java/Others)&#160;&#160;&#160;&#160;Memory Limit: 65536/32768 K (Java/Others)<br>Total Submission(s): 30&#160;&#160;&#160;&#160;Accepted Submission(s): 3<br></span></b></font><br><br><div class="panel_title" align="left">Problem Description</div> <div class="panel_content">After long studying how embryos of organisms become asymmetric during their development, Dr. Podboq, a famous biologist, has reached his new hypothesis. Dr. Podboq is now preparing a poster for the coming academic conference, which shows a tree representing the development process of an embryo through repeated cell divisions starting from one cell. Your job is to write a program that transforms given trees into forms satisfying some conditions so that it is easier for the audience to get the idea.<br><br>A tree representing the process of cell divisions has a form described below.<br><br>    * The starting cell is represented by a circle placed at the top.<br>    * Each cell either terminates the division activity or divides into two cells. Therefore, from each circle representing a cell, there are either no branch downward, or two branches down to its two child cells. <br><br>Below is an example of such a tree.<br><img src="../../../data/images/2314-1.png"><br>Figure F-1: A tree representing a process of cell divisions<br><br>According to Dr. Podboq's hypothesis, we can determine which cells have stronger or weaker asymmetricity by looking at the structure of this tree representation. First, his hypothesis defines "left-right similarity" of cells as follows:<br><br>   1. The left-right similarity of a cell that did not divide further is 0.<br>   2. For a cell that did divide further, we collect the partial trees starting from its child or descendant cells, and count how many kinds of structures they have. Then, the left-right similarity of the cell is defined to be the ratio of the number of structures that appear both in the right child side and the left child side. We regard two trees have the same structure if we can make them have exactly the same shape by interchanging two child cells of  some cells. <br><br>For example, suppose we have a tree shown below:<br><img src="../../../data/images/2314-2.png"><br>Figure F-2: An example tree<br><br>The left-right similarity of the cell A is computed as follows. First, within the descendants of the cell B, which is the left child cell of A, the following three kinds of structures appear. Notice that the rightmost structure appears three times, but when we count the number of structures, we count it only once. <br><img src="../../../data/images/2314-3.png"><br>Figure F-3: Structures appearing within the descendants of the cell B<br><br>On the other hand, within the descendants of the cell C, which is the right child cell of A, the following four kinds of structures appear. <br><img src="../../../data/images/2314-4.png"><br><br>Figure F-4: Structures appearing within the descendants of the cell C<br><br>Among them, the first, second, and third ones within the B side are regarded as the same structure as the second, third, and fourth ones within the C side, respectively. Therefore, there are four structures in total, and three among them are common to the left side and the right side, which means the left-right similarity of A is 3/4.<br><br>Given the left-right similarity of each cell, Dr. Podboq's hypothesis says we can determine which of the cells X and Y has stronger asymmetricity by the following rules.<br><br>   1. If X and Y have different left-right similarities, the one with lower left-right similarity has stronger asymmetricity.<br>   2. Otherwise, if neither X nor Y has child cells, they have completely equal asymmetricity.<br>   3. Otherwise, both X and Y must have two child cells. In this case, we compare the child cell of X with stronger (or equal) asymmetricity (than the other child cell of X) and the child cell of Y with stronger (or equal) asymmetricity (than the other child cell of Y), and the one having a child with stronger asymmetricity has stronger asymmetricity.<br>   4. If we still have a tie, we compare the other child cells of X and Y with weaker (or equal) asymmetricity, and the one having a child with stronger asymmetricity has stronger asymmetricity.<br>   5. If we still have a tie again, X and Y have completely equal asymmetricity. <br><br>When we compare child cells in some rules above, we recursively apply this rule set.<br><br>Now, your job is to write a program that transforms a given tree representing a process of cell divisions, by interchanging two child cells of arbitrary cells, into a tree where the following conditions are satisfied.<br><br>   1. For every cell X which is the starting cell of the given tree or a left child cell of some parent cell, if X has two child cells, the one at left has stronger (or equal) asymmetricity than the one at right.<br>   2. For every cell X which is a right child cell of some parent cell, if X has two child cells, the one at right has stronger (or equal) asymmetricity than the one at left. <br><br>In case two child cells have equal asymmetricity, their order is arbitrary because either order would results in trees of the same shape.<br><br>For example, suppose we are given the tree in Figure F-2. First we compare B and C, and because B has lower left-right similarity, which means stronger asymmetricity, we keep B at left and C at right. Next, because B is the left child cell of A, we compare two child cells of B, and the one with stronger asymmetricity is positioned at left. On the other hand, because C is the right child cell of A, we compare two child cells of C, and the one with stronger asymmetricity is positioned at right. We examine the other cells in the same way, and the tree is finally transformed into the tree shown below. <br><img src="../../../data/images/2314-5.png"><br>Figure F-5: The example tree after the transformation<br><br>Please be warned that the only operation allowed in the transformation of a tree is to interchange two child cells of some parent cell. For example, you are not allowed to transform the tree in Figure F-2 into the tree below. <br><img src="../../../data/images/2314-6.png"><br>Figure F-6: An example of disallowed transformation </div>
<div class="panel_bottom">&#160;</div>
<br><div class="panel_title" align="left">Input</div> <div class="panel_content">The input consists of n lines (1≤n≤100) describing n trees followed by a line only containing a single zero which represents the end of the input. Each tree includes at least 1 and at most 127 cells. Below is an example of a tree description.<br><br>    ((x (x x)) x) <br><br>This description represents the tree shown in Figure F-1. More formally, the description of a tree is in either of the following two formats.<br><br>    "(" &lt;description of a tree starting at the left child&gt; &lt;single space&gt; &lt;description of a tree starting at the right child&gt; ")" <br><br>or<br><br>    "x" <br><br>The former is the description of a tree whose starting cell has two child cells, and the latter is the description of a tree whose starting cell has no child cell.</div>
<div class="panel_bottom">&#160;</div>
<br><div class="panel_title" align="left">Output</div> <div class="panel_content">For each tree given in the input, print a line describing the result of the tree transformation. In the output, trees should be described in the same formats as the input, and the tree descriptions must appear in the same order as the input. Each line should have no extra character other than one tree description.</div>
<div class="panel_bottom">&#160;</div>
<br><div class="panel_title" align="left">Sample Input</div>
<div class="panel_content"><pre><div style="font-family:Courier New,Courier,monospace;">(((x x) x) ((x x) (x (x x))))
(((x x) (x x)) ((x x) ((x x) (x x))))
(((x x) ((x x) x)) (((x (x x)) x) (x x)))
(((x x) x) ((x x) (((((x x) x) x) x) x)))
(((x x) x) ((x (x x)) (x (x x))))
((((x (x x)) x) (x ((x x) x))) ((x (x x)) (x x)))
((((x x) x) ((x x) (x (x x)))) (((x x) (x x)) ((x x) ((x x) (x x)))))
0</div></pre></div>
<div class="panel_bottom">&#160;</div>

<div class="panel_content"><pre><div style="font-family:Courier New,Courier,monospace;">((x (x x)) ((x x) ((x x) x)))
(((x x) ((x x) (x x))) ((x x) (x x)))
(((x ((x x) x)) (x x)) ((x x) ((x x) x)))
(((x ((x ((x x) x)) x)) (x x)) ((x x) x))
((x (x x)) ((x (x x)) ((x x) x)))
(((x (x x)) (x x)) ((x ((x x) x)) ((x (x x)) x)))
(((x (x x)) ((x x) ((x x) x))) (((x x) (x x)) (((x x) (x x)) (x x))))</div></pre></div>
